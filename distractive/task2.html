Summary

Users report that discount codes initially appear successfully applied but are later removed during checkout validation.

Description

Customer support escalated a newly reported issue affecting multiple users during checkout. Users apply a promotional code and initially receive a success confirmation. Shortly afterwards the discount is removed from the applied list, reappears in the available list, and the final payable amount remains unchanged. The issue has been reproduced across multiple vendors.

Observed Behaviour
  • Green success ribbon appears immediately after code entry
  • Applied code is later removed from the active promotion area
  • The same code returns to the available promotion list
Impact

Users experience confusion during checkout and may abandon the purchase flow due to inconsistent promotion feedback.
